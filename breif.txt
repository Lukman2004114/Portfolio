## 1. CONCEPT OVERVIEW

**What is it?**
A personal portfolio website for Lukman Al Afandi — a Cybersecurity student at Gannon University (Class of 2030) in Erie, PA. The site is a professional landing page designed to impress recruiters, professors, and potential collaborators. It showcases who Lukman is, what he knows, and what he has built.

**The audience:**
- Recruiters and hiring managers at cybersecurity firms
- Internship coordinators
- Professors and academic connections
- Fellow students and CTF community members
- Anyone who Googles his name

**The goal:**
When someone visits, they should immediately understand:
1. Who Lukman is and where he studies
2. What areas of cybersecurity he specializes in
3. What he has actually built and done
4. How to contact him or download his resume

**The feel:**
Professional but not corporate. Technical but not cold. It should feel like a developer who takes their craft seriously — someone who built a home lab for fun, not because they had to. The hacker-terminal aesthetic runs throughout: boot screen, decrypt animations, typing effects, matrix green dark mode.

## 3. BRAND VOICE

**Personality:** Technically sharp, curious, direct. Not arrogant — confident. Writes like a developer who knows what they're talking about but doesn't feel the need to prove it. Occasionally uses hacker culture references naturally, not as performance.

**Tone:**
| Dimension | Score | Notes |
|-----------|-------|-------|
| Formal ↔ Casual | 3/5 | Professional but human |
| Serious ↔ Playful | 2.5/5 | Serious work, light touch |
| Reserved ↔ Expressive | 3/5 | Clear and direct |
| Technical ↔ Accessible | 3/5 | Technical terms used correctly, never jargon for jargon's sake |

**Voice rules:**
- First person always: "I built," "I'm studying," "I love" — never "this developer" or "Lukman has"
- Active voice. Short sentences. Get to the point.
- Technical terms are fine — this audience knows what Metasploit is
- Don't oversell. "I'm learning X" is more credible than "I'm an expert in X" for a student
- Curiosity is the core trait. Lead with it.

**Words to use:** built, explored, breaking, secure, hands-on, practical, curious, challenge, lab, defense
**Words to avoid:** passionate about (overused), world-class, expert, guru, ninja, leverage, synergy

**Core messaging:**
- **Headline:** "Lukman Al Afandi"
- **Sub-role (typing effect, rotates):** Cybersecurity Student · Ethical Hacker · CTF Enthusiast · Network Defender · Penetration Tester
- **Hero description:** "Passionate about ethical hacking, network defense, and building secure digital systems."
- **About opening:** "I love breaking things apart to understand how they work — and then building them back stronger."
- **Contact CTA:** "I'm always open to discussing cybersecurity projects, internship opportunities, or just talking about the latest vulnerabilities."

## 5. ANIMATIONS & INTERACTIONS

All animations are built in vanilla JavaScript (no libraries). Every animation respects `prefers-reduced-motion` (not yet implemented — add this).

### On Load

**Boot Screen** (plays once per browser session):
```
[LA/]$ boot --portfolio --secure
> Initializing modules...
> Loading assets ............. [OK]
> Establishing connection ..... [OK]
> Running security checks ..... [OK]
> Access granted. Welcome, visitor.
```
Terminal font, green text on near-black background. Fades out after ~3 seconds. Skipped on return visits (sessionStorage flag).

**Page fade-in:** Body fades from opacity 0 → 1 over 0.6s on load.

**Hero glitch effect:** The name "Al Afandi" runs a glitch animation once, 1.2s after load. Cyan + purple text-shadow displacement. Lasts 1.5s then stops.

### On Scroll

**Scroll reveal:** Every section element with `.reveal` fades in + rises 28px when it enters the viewport. Uses `IntersectionObserver` at threshold 0.12, with 48px root margin. Fires once only.

**Stagger delays on cards:**
- 1st card: 0ms · 2nd: 80ms · 3rd: 160ms · 4th: 240ms · 5th: 320ms · 6th: 400ms

**Decrypt text:** Section headings with `.decrypt` class run a character-scramble animation when they enter view. Characters resolve left-to-right from a pool of `!<>-_\/[]{}—=+*^?#@$%ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789`. Applied to: "My Skills" and "Recent Projects" headings.

**Active nav link:** The navbar highlights the current section's link as you scroll (IntersectionObserver at threshold 0.4).

**Scroll progress bar:** 3px bar at the very top of the page. Fills left-to-right as you scroll. Gradient: cyan → sky → purple. `z-index: 9999`.

**Counter animation:** Stats (15+, 5+, 3+) count up from 0 when the About section scrolls into view. Eased with `easeOutQuad` over 1.6s.

### On Hover (desktop only)

**Card 3D tilt:** Project and skill cards tilt up to ±5° on X/Y axis based on mouse position within the card. `perspective(600px)`. Resets on `mouseleave`.

**Skill card scan line:** A colored line sweeps top-to-bottom across skill cards on hover over 0.7s.

**Project card shimmer:** A light gradient line sweeps left-to-right across the top edge of project cards on hover.

**CTA buttons:** `translateY(-2px)` + shadow increase on hover, 0.3s.

**Cursor glow (desktop):** A 300×300px radial gradient follows the cursor with a 0.18s lag. Cyan core, fades to transparent. Disappears after 1.2s of no movement.

### Terminal Typing Effect

The `.role-badge` in the hero cycles through 5 roles:
1. Cybersecurity Student
2. Ethical Hacker
3. CTF Enthusiast
4. Network Defender
5. Penetration Tester

Types at 85ms/char, deletes at 50ms/char. 2.4s pause when fully typed. Prefixed with `> ` terminal prompt. Blinking cursor `_`.

### Dark Mode

Toggle button in navbar (moon/sun icon). Preference saved to `localStorage`. Applies `data-theme="dark"` to `<html>`. Instant switch, no transition delay.

## 9. UI UX PRO MAX — Design System Audit

Ran a full design system analysis against the portfolio. Here are the findings.

### Style Direction Name
**"Dark Mode OLED Portfolio with Terminal Aesthetic"**
- High contrast, deep black, eye-friendly for night use
- Best For: coding platforms, developer tools, hacker-culture portfolios, OLED devices
- Performance: ⚡ Excellent | Accessibility: ✓ WCAG AAA capable

### Font Upgrade Recommendation

Current font (Poppins) is good but generic. UI UX Pro Max matched the portfolio to the **Developer Mono** pairing:

| Role | Font | Why |
|------|------|-----|
| **Headings** | `JetBrains Mono` | Terminal/hacker feel, code-native, distinctive — perfect for a security portfolio |
| **Body** | `IBM Plex Sans` | Professional, highly readable, developer-native at IBM |

**CSS import to replace Poppins:**
```css
@import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500;600;700&family=IBM+Plex+Sans:wght@300;400;500;600;700&display=swap');
```

> Note: This is an upgrade suggestion, not a requirement. Poppins works fine. JetBrains Mono headings would push the hacker aesthetic further and make the portfolio more distinctive.

### Color Validation

UI UX Pro Max independently matched the dark mode palette to the **Cybersecurity Platform** preset — validating the existing color choices:

| Role | Current hex | Preset hex | Match |
|------|------------|------------|-------|
| Primary / Accent | `#4ade80` | `#00FF41` | ✅ Matrix green — correct |
| Background | `#080d08` | `#000000` | ✅ Near-black — correct |
| Card surface | `#101810` | `#0C130E` | ✅ Dark card — correct |
| Border | `#1e331e` | `#1F1F1F` | ✅ Subtle dark border — correct |

Light mode uses cyan `#0891b2` — UI UX Pro Max classifies this as a professional developer/SaaS palette. No changes needed.

### UX Issues Flagged

These were identified as missing or incomplete:

| Issue | Priority | Fix |
|-------|----------|-----|
| `prefers-reduced-motion` not honored | HIGH | Add `@media (prefers-reduced-motion: reduce)` to CSS — collapse all transforms to opacity-only at 200ms |
| No visible focus rings on interactive elements | HIGH | Add 2px cyan focus ring with 2px offset on all buttons, links, inputs |
| `cursor: pointer` missing on some clickable elements | MEDIUM | Add to skill cards and any div-based clickable elements |
| Profile image has no `width`/`height` attributes | MEDIUM | Causes CLS (layout shift) — add explicit dimensions |
| No `<main>` landmark wrapping sections | MEDIUM | Wrap all `<section>` elements in `<main>` for screen readers |
| No skip-to-content link | LOW | Add hidden `<a href="#main">Skip to content</a>` at top of `<body>` |
